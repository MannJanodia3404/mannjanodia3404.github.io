About Me

Why I Build Robots

From Curiosity to Engineering

I am a robotics engineer specializing in full-stack architecture, focusing on the seamless integration of mechanical hardware, control software, and artificial intelligence. I am driven by the moment these disciplines converge, whether it's developing autonomous navigation systems or designing custom neural networks for adaptive flight simulations. I don't just write code; I build complete systems that turn complex technical challenges into functional, real-world solutions.

I am currently pursuing my MSc in Robotics Engineering at Middlesex University, building upon my BEng in Mechatronics with a minor specialization in AI and Machine Learning. My background includes developing award-winning, patent-pending self-balancing systems and vision-based specimen handling platforms using high-precision depth sensing. I am particularly interested in narrowing the gap between "what robots can do" and "what we need them to do" through rigorous validation and system optimization.

When I'm not in the lab, you can find me on the field playing competitive cricket, football, or volleyball. I also enjoy the strategic challenge of a chess match and reading new books.

What drives me? The moment hardware, software, and intelligence converge perfectly.

Gesture Multi-Drone Control

AI · Custom NN · Webots · Robotics

Overview

A fully gesture-driven system to control 3 simulated DJI Mavic2Pro drones in Webots using only hand gestures captured by a laptop webcam. Two separate neural networks, both built from scratch in NumPy, to classify left-hand pose (which drone to select) and right-hand pose (what action to perform), with results written to a shared JSON file that each drone's controller reads every 8ms.

Key Achievements

  • Two-hand control: Left hand selects drone (1/2/3 fingers or open palm for all). Right hand commands: Takeoff, Land, Altitude Up/Down, Flip Forward, Hover.
  • Custom 5-layer FC-NN from scratch: 35 → 64(ReLU) → 32(ReLU) → 16(Tanh) → N(Softmax), L2 regularisation, cross-entropy loss. No PyTorch, no TensorFlow.
  • Three optimisers: Adam, L-BFGS, and Nelder-Mead compared with 5-fold cross-validation.
  • 35-feature hand vector from MediaPipe HandLandmarker: 10 fingertip positions, 5 extension angles, 3 wrist orientations, 2 palm offsets.
  • Flip state machine: Windup 160ms → Rotate 320ms → Recover 240ms for safe forward flips above 1.5m altitude.
  • Fail-safe: No gesture for >2s causes all drones to hold position autonomously.
  • Personally recorded 800 to 1,200 training samples across 10 gesture classes.

Camera Servo Tracking (RPi)

Computer Vision · Embedded

Overview

Real-time multi-object detection and tracking on Raspberry Pi 4 using YOLOv8 (NCNN-exported) with two MG996R servos for pan-tilt camera control.

Key Achievements

  • Picamera2 and YOLOv8n NCNN model for low-latency on-device inference without a GPU.
  • Proportional control keeps the selected class centred in frame with configurable deadband, max step angle, and gain.
  • External 5V servo power supply prevents Pi brownout under servo load.
  • pigpio PWM on GPIO17 for pan and GPIO18 for tilt.

Autonomous Mapping Robot

Embedded · C++ · Robotics

Overview

A dual-Arduino autonomous robot that maps a room using HC-SR04 ultrasonic sensors and encoder odometry, then executes a planned return journey with 5 equally-spaced scheduled stops. Clean Master/Slave architecture separates sensing from motor control across two Arduino Uno boards.

Key Achievements

  • Master board sends MAP, RUN, STOP, SLOW commands to the Slave motor board via serial.
  • Encoder odometry: 850 counts per revolution, minimum detectable movement approximately 7 micrometres.
  • Bridge detection via upward HC-SR04: auto-slows to 50% speed through narrow passages.
  • Rear obstacle safety: emergency stop with red LED on unexpected obstacle.
  • DRV8833 motor driver with tunable LEFT_SPEED_CORRECTION for drift compensation.
  • Configurable stop count via NUM_INTERVALS constant, no re-wiring needed.

Niryo NED2 Test Tube Sorting System

Robotics · RFID · ROS2 · Embedded

Overview

A fully autonomous three-component lab robotics system combining mobile navigation, 6-DOF manipulation, and RFID identification to sort test tubes without human intervention.

Key Achievements

  • Three-layer architecture: Hardware (NED2 + TurtleBot4 + ESP32), Software (ROS2 + PyNiryo + Flask), Integration (HTTP REST API + threading).
  • Flask REST API with 6 endpoints: /start_niryo, /rfid, /niryo_done, /status, /emergency_stop, /reset.
  • Event-driven RFID sync: threading.Event() blocks Niryo at scanner until ESP32 sends tag UID via WiFi HTTP GET.
  • 4-bin Cartesian sorting at SLOW_SPEED (20%) for safe tube insertion with joint-space waypoints.
  • ESP32 + PN532 RFID: I2C on GPIO21/22, MIFARE ISO14443A, under 5cm range.
  • TurtleBot4 coordinator: navigate, dock, trigger Niryo, poll status, undock, repeat.
